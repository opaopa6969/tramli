## 設計判断

- **DD-041**: FlowContext snapshot 案B 採用（`instance.context.snapshot()` + `FlowContextSnapshot` 型）
- **DD-042**: FlowProvider 1エンジン前提・ネスト禁止
- **DD-043**: useFlowChain は React 層に置く（`when` シグネチャ: `(prevState, prevContext, prevError?)` + context コピー渡し）
- **DD-044**: FlowProvider は `sessionId` prop で再マウント時のフロー復元を可能にする（永続化・自動推論はしない）
- **DD-045**: `isLoading` 中の `resume` 呼び出しは黙って無視（throw せず、キューイングもしない）
- **DD-046**: UI 更新は `startFlow` / `resumeAndExecute` 完了後のみ — 中間 `put()` は届かない仕様として明文化（README/JSDoc 記載）
- **DD-047**: hierarchy plugin の LCA ランタイム実装を凍結、`#8` はクローズ（DD-021 矛盾・ユースケース不在・メンテ負債）
